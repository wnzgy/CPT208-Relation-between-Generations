Prototype Structure

Clear end-to-end flow from keepsake entry to shared memory revisits

Step 1

Role Entry

User opens the app and selects the storyteller identity.

Step 2

Tap / Scan Trigger

NFC-first entry, with a manual scan fallback path for unsupported devices.

Step 3

3D Memory Reveal

A keepsake unlocks the story card and visual memory scene.

Step 4

Reply Loop

Cross-generation replies keep the memory active as dialogue, not archive.

Step 5

Memory Collection

Users revisit memories through map and collection states over time.

Core Loop:

Role select -> scan keepsake -> reveal memory -> reply -> collect and revisit.

Implementation Highlights

What reviewers should know about reliability and usability

Performance

  • 3D canvas tuned for mobile rendering budget
  • Smooth transition states to avoid abrupt context switch
  • Local-first data to keep interaction responsive

Compatibility

  • Web NFC path for supported Android browsers
  • Manual scan fallback for unsupported environments
  • Touch targets sized for classroom demo reliability

Risk and Mitigation

Risk: NFC capability differs by device model.
Mitigation: Fallback unlock path included in prototype.

Risk: 3D rendering overhead on low-end devices.
Mitigation: Controlled model complexity and simpler effect stack.
